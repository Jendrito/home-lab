1. Para el análisis centralizado y automatizado de logs usaremos Wazuh, software que podríamos utilizarlo en Kali, pero debido a que más facil de manejar y estable en Ubuntu Server, utilizaremos este sistema operativo. Hay dos maneras de hacerlo: creando una máquina virtual o, desde Kali, levantar un contenedor basado en Ubuntu Server usando Docker y, luego, instalar Wazuh dentro de ese contenedor. En mi caso, voy a usar la segunda opción y, de esta manera, incorporar nuevos conceptos dentro del home-lab.

2. El primer paso será correr una serie de comandos para actualizar Kali, instalar dependencias necesarias, agregar el repositorio Docker, instalar Docker Engine y Docker Compose, y verificar que todo se haya instalado correctamente. Para este paso, cree un subcarpeta llamada "Instalación Docker en Kali" con su respectivo archivo .txt y y los prints del proceso.

3. El segundo paso ha implementar será instalar Wazuh Manager, que será el motor central de la plataforma. Va a encargarse de recibir los logs que nos envíe Windows (o las máquinas virtuales que se deseen), analizarlos, aplicar reglas de correlación y detección, y, finalmente, poder enviar los eventos a otras herramientas de visualización como Kibana. Similiar al paso anterior, hay una subcarpeta llamada "Instalación Wazuh Manager en Ubuntu".
